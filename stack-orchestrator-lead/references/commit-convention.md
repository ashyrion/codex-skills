# Commit Convention (Required)

## Message format
- `<type>: <한국어 요약>`

## Allowed types
- `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `perf`, `ci`, `build`, `revert`

## Rules
- Type prefix is mandatory.
- Summary must be written in Korean.
- Keep the first line concise and action-oriented.

## Examples
- `feat: 스케줄 배정 우선순위 로직 추가`
- `fix: 휴무일 편집 가능 버그 수정`
- `docs: 프로젝트 프로파일 작성 가이드 보강`
